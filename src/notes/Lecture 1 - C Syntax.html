realloc

  • Allows you to change the size of malloc'd memory

  • realloc(ptr, size)

    • where ptr is the pointer to the malloc'd memory

  • Returns a pointer to the newly allocated memory or NULL if it did not have enough memory

  • Size is what you want the new size of the memory to be

    • e.g. previously was 5 * sizeof(int), and you want to increase it, just give the new size

      • e.g. 6 * sizeof(int) gives original 5 + 1 extra int

Debuggers

  • Using gdb
  1. Compile with -g tag (gcc -o file file.c -g)
  2. Run program using gdb file
  3. Use whatever commands needed

  • Commands

    • run ARGS// run the program with given ARGS
    • set args ARGS // set command args
    • break EXPR // set a breakpoint (pause in the program) at a given line number or function
    • step, next, until, continue - all are used to continue the program after a break point
    • print EXPR // print a given variable
    • many more commands: https://web.eecs.umich.edu/~sugih/pointers/summary.html

Stacks

  • FIFO (First in, First Out)

    • Web browser history
    • Balanced Bracket Checking
    • HTML Tag matching
    • Functions

  • Push

    • Add item to top of stack

  • Pop

    • Remove top most item

  • Size

    • Number of items in stack

  • Peek

    • Get top most item without removing

    • make a new empty stack

  • Destructor

    • Release entire stack

 

Infix/Postfix/Prefix

  • Infix

    • a + b

  • Prefix

    • +a b
    • If you see an operator, conduct that operator on the next 2 numbers

  • Postfix

    • a b +
    • If you see an operator, conduct that operation on previous 2 numbers

 

Queues

  • FIFO

    • first in, first out

  • Enqueue

    • Add new item to end of queue (back of queue)

  • Dequeue

    • Remove item from front of queue

  • Size

    • Return number of items in queue

  • Peek

    • Return item from front of queue without removing it

  • Constructor/Destructor

    • create/destroy the queue

  • Keep a pointer to head and tail of the list

 

Analysis and Testing

  • Good Software:

    • Correctness:

      • Returns expected result for all valid inputs

    • Efficiency

      • Returns results reasonably quickly

  • Many more, but irrelevant for this course

 

Analysing Effectiveness

  • Empirical

    • testing via program execution
    • Make test cases and compare to expected results

  • Theoretical

    • Proof of program correctness
    • Define pre-/post-conditions
    • Establish that code maps from pre- to post-

 

  • e.g. find the max value in an unsorted array

  • Empirical:

    • Test if max value is first/last/middle
    • Values are positive/negative/mixed

  • Theoretical

    • Pre-conditions

      • for all j in 0...N, defined(a[j])

        • i.e. all numbers in the array are defined

    • Post-conditions

      • For all j in 0...N

        • Max number a[j]
        • i.e. max number is the biggest number

 

Testing Approaches

  • Test-Driven Development (TDD)/Test first

    • Create tests before writing the function
    • Then write the function and test it

 

  • Regression Testing

    • Keep a comprehensive testing suite
    • Always re-run tests after changing the system

 

  • All tests:

    • Create a well-known environment
    • Mutate 1 thing only
    • Check results

 

Black-Box Testing

  • Checks input/output only
  • Implementation is irrelevant

 

White-Box Testing

  • Test code from the inside
  • Check code structure and consistency
  • Checks the functions
  • Relies on specific implementations
